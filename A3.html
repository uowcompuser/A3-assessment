Web Attacks Table

Name Web Attacks
SQL Injection Source: https://www.sitepoint.com/

SQL Injection is a code injection technique that exploits a vulnerability in an application's software where it allows attackers to manipulate database queries and usually occurs when you ask a user for input, like their username/userid, and instead of a name/id, the user gives you an SQL statement that you will unknowingly run on your database.

Source: https://www.w3schools.com/sql/sql_injection.asp

XSS (Cross-Site Scripting) Source: https://evalian.co.uk/xxs-attacks-what-is-cross-site-scripting-and-why-is-it-a-security-risk/

An attacker can use XSS to send a malicious script to an unsuspecting user where the end user’s browser has no way to know that the script should not be trusted, and will execute the script because it thinks the script came from a trusted source, the malicious script can access any cookies, session tokens, or other sensitive information retained by the browser and used with that site.

Source: https://owasp.org/www-community/attacks/xss/
Broken Authentication Source: https://prophaze.com/web-application-firewall/what-is-broken-authentication/

Broken authentication or any sort of illegitimate login–based access can be executed in many ways, such as with brute forcing passwords, credential stuffing, with dictionary attacks and in many other ways. There are many ways to gain access to accounts however a long and sophisticated password is greatly beneficial to combat these attacks, as well as using two factor authentication.

Source: https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

Man in the middle attacks Source: https://threatcop.com/blog/man-in-the-middle-attack/

Man in the middle attacks are common among sites that have not encrypted their data as it travels from the user to the servers (sites using HTTP instead of HTTPS) and also is completed by the threat actor who intercepts the data as it is being transferred between two parties. If the data isn't encrypted, the attacker can easily read personal, login, or other sensitive details that travel between two locations on the Internet.

Source: https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself#:~:text=injection%20or%20XSS.-,9,-.%20MiTM%20(Man%2Din

Directory Traversal Source: https://www.insecure.in/directory-traversal-attack

A path traversal attack (also known as directory traversal) aims to access files and directories that are stored outside the web root folder through manipulating variables that reference files with “dot-dot-slash (../)” sequences and its variations or by using absolute file paths, it may be possible to access arbitrary files and directories stored on file system including application source code or configuration and critical system files.

Source: https://owasp.org/www-community/attacks/Path_Traversal

Unicode encoding Source: https://hackaday.com/2013/09/27/utf-8-the-most-elegant-hack/

The attack aims to explore flaws in the decoding mechanism implemented on applications when decoding Unicode data format where an attacker can use this technique to encode certain characters in the URL to bypass application filters, thus accessing restricted resources on the Web server or to force browsing to protected pages.

Source: https://owasp.org/www-community/attacks/Unicode_Encoding

Repudiation attack Source: https://softwareg.com.au/blogs/internet-security/non-repudiation-in-network-security

A repudiation attack occurs when a user or system denies having performed a particular action or transaction, thereby undermining accountability and traceability through exploiting the absence of proper tracking and logging mechanisms, allowing malicious actors to manipulate or forge the identification of actions. As a result, the data stored in log files can become invalid or misleading, making it difficult to establish responsibility for specific activities.

Source: https://www.twingate.com/blog/glossary/repudiation%20attack

Denial of Serive (DOS) Source: https://www.researchgate.net/figure/Denial-of-Service-DoS-attack-A-DDoS-Distributed-Denial-of-Service-attack-as-shown-in_fig2_324562008

A denial-of-service (DoS) attack is a type of cyber attack in which a malicious actor aims to render a computer or other device unavailable to its intended users by interrupting the device's normal functioning through temporarily or indefinitely disrupting services of a hose connected to a network through using multiple computers or machines to flood a targeted resource.

Source: https://owasp.org/www-community/attacks/Denial_of_Service

Trojan Horse Source: https://www.shiksha.com/online-courses/articles/trojan-horse-attack/

A Trojan Horse is a program that uses malicious code masqueraded as a trusted application. The malicious code can be injected on benign applications, masqueraded in e-mail links, or sometimes hidden in JavaScript pages to make furtive attacks against vulnerable internet Browsers.

Source: https://owasp.org/www-community/attacks/Trojan_Horse.org/www-community/attacks/Denial_of_Service

Spyware Source: https://www.chargebackgurus.com/blog/spyware

Spyware is a program that captures statistical information from a user’s computer and sends it over internet without user acceptance. This information is usually obtained from cookies and the web browser’s history. Spyware can also install other software, display advertisements, or redirect the web browser activity. Spyware differs from a virus, worm, and adware in various ways. Spyware does not self-replicate and distribute itself like viruses and worms, and does not necessarily display advertisements like adware.

Source: https://owasp.org/www-community/attacks/Spyware
